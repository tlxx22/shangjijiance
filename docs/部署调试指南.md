## 构建流程

### 技术栈

| 组件 | 说明 |
|-----|------|
| 依赖管理 | uv + pyproject.toml |
| 构建方式 | uv sync |
| 浏览器 | Playwright Chromium（运维预装） |
| 运行时 | `deploy/entrypoint.sh`（默认 `nginx + uvicorn`，可回滚到 `legacy_gunicorn`） |
| 基础镜像 | crawler-agent-base:stable（含 Chrome + 中文字体） |

### 构建步骤

1. `apt-get install nginx supervisor` + `uv sync` - 安装运行时依赖和 Python 依赖
2. `deploy/entrypoint.sh` - 统一处理浏览器软链接修复、启动通知、启动模式切换

### 三一环境特别说明

- 三一环境的代码主体与本地仓库保持一致，但云桌面上的 `trans.py` 为三一专用版本
- 从本地上传压缩包覆盖云桌面代码时，必须继续沿用当前流程：**排除本地 `trans.py`**，保留云桌面已有的三一专用 `trans.py`
- 本地 `trans_sany.py` 仅作对照参考，不参与运行，也不参与部署自动切换

## 常见问题与解决方案

### 1. uv pip install 报错：需要虚拟环境

**错误信息**：

```
error: Failed to install... Using Python at /usr/local/bin/python, which is not a virtual environment
```

**原因**：uv 默认需要在虚拟环境中安装，不支持 `--system` 全局安装。

**解决方案**：使用 `uv sync` 替代 `uv pip install`，它会自动创建/使用虚拟环境。

---

### 2. hatchling 打包失败：找不到包目录

**错误信息**：

```
ValueError: Unable to determine which files to ship inside the wheel
The most likely cause is that there is no directory that matches the name of your project (crawler_agent)
```

**原因**：项目结构不符合 hatchling 默认规则（没有 `crawler_agent/` 目录）。

**解决方案**：在 `pyproject.toml` 中显式配置打包内容：

```toml
[tool.hatch.build.targets.wheel]
packages = ["src", "app.py", "gunicorn.conf.py", "prompts", "extract_fields.yaml"]
```

---

### 3. 浏览器找不到：No local browser binary found

**错误信息**：

```
ERROR [BrowserSession] [LocalBrowserWatchdog] ⚠️ No local browser binary found, installing browser using playwright subprocess...
```

**原因**：

1. Playwright 浏览器安装在非默认路径
2. 目录名不匹配（`chrome-linux64` vs `chrome-linux`）
3. 环境变量 `PLAYWRIGHT_BROWSERS_PATH` 未生效

**解决方案**：在统一启动入口 `deploy/entrypoint.sh` 中创建软链接：

```bash
# 1. 链接浏览器目录到默认路径
mkdir -p /root/.cache/ms-playwright
ln -sf /mnt/.devops_uv_cache/venv/browsers/* /root/.cache/ms-playwright/

# 2. 修复目录名不匹配问题
ln -sf /root/.cache/ms-playwright/chromium-1200/chrome-linux64 /root/.cache/ms-playwright/chromium-1200/chrome-linux
```

---

### 4. API Key 无效

**错误信息**：

```
Invalid API key. Get your API key at https://cloud.browser-use.com/new-api-key
```

**排查步骤**：

1. 检查环境变量是否设置：

   ```bash
   echo $BROWSER_USE_API_KEY
   echo "${#BROWSER_USE_API_KEY}"  # 检查长度
   ```

2. 确认 Key 是否有空格/换行符：

   ```bash
   echo "|$BROWSER_USE_API_KEY|"
   ```

3. 确认 Jenkinsfile 中的 Key 与运维设置的一致

**解决方案**：让运维通过 K8s Secret 设置环境变量，避免代码中硬编码。

---

### 5. 新模型不支持

**错误信息**：

```
ValueError: Invalid model: 'bu-30b-a3b-preview'. Must be one of ['bu-latest', 'bu-1-0']
```

**原因**：browser-use 源码中硬编码了支持的模型列表。

**解决方案**：在构建时用 sed 修改源码：

```bash
sed -i "s/valid_models = \['bu-latest', 'bu-1-0'\]/valid_models = ['bu-latest', 'bu-1-0', 'bu-30b-a3b-preview']/g" \
  $UV_PROJECT_ENVIRONMENT/lib/python3.11/site-packages/browser_use/llm/browser_use/chat.py
```

---

### 6. Groovy 变量展开问题

**问题**：Jenkinsfile 中 `$UV_PROJECT_ENVIRONMENT` 被 Groovy 解析为空。

**原因**：双引号字符串中 `$` 会被 Groovy 当作变量展开。

**解决方案**：转义 `$` 为 `\$`：

```groovy
configMap.put("BUILD_COMMAND", "... \$UV_PROJECT_ENVIRONMENT/...")
```

## 环境变量配置

### 运维需配置的环境变量

| 变量名 | 说明 | 示例值 |
|-------|------|-------|
| `BROWSER_USE_API_KEY` | Browser-use API 密钥 | `bu_xxx...` |
| `PLAYWRIGHT_BROWSERS_PATH` | 浏览器安装路径 | `/mnt/.devops_uv_cache/venv/browsers` |
| `UV_PROJECT_ENVIRONMENT` | uv 虚拟环境路径 | `/mnt/.devops_uv_cache/venv` |

### 可选环境变量

| 变量名 | 说明 | 默认值 |
|-------|------|-------|
| `WORKERS` | 内部 worker 数量 | `5` |
| `TIMEOUT_SECONDS` | 单站点超时时间（秒） | `1800` |
| `SERVER_MODE` | `nginx_uvicorn` / `legacy_gunicorn` | `nginx_uvicorn` |
| `BROWSER_USE_DAILY_BUDGET_USD` | browser-use 日预算上限（USD） | `50` |

## Jenkinsfile 配置说明

```groovy
// 构建命令
configMap.put("BUILD_COMMAND", "apt-get update && apt-get install -y --no-install-recommends nginx supervisor && rm -rf /var/lib/apt/lists/* && uv sync")

// 启动命令（统一入口，默认 SERVER_MODE=nginx_uvicorn）
configMap.put("APP_RUN_COMMAND", "sh -c 'exec sh /app/deploy/entrypoint.sh'")

// 基础镜像（预装 Chrome + 中文字体）
configMap.put("DOCKERFILE_BASE_IMAGE", "yaowu-registry-vpc.cn-shanghai.cr.aliyuncs.com/pulic/crawler-agent-base:stable")
```

## 正式环境部署检查清单

- [ ] 确认基础镜像版本正确
- [ ] 确认运维已配置 `BROWSER_USE_API_KEY` 环境变量
- [ ] 确认运维已执行 `playwright install-deps chromium`
- [ ] 确认 `/mnt/.devops_uv_cache/venv/browsers/` 路径存在
- [ ] 确认 `SERVER_MODE` 未设置时默认走 `nginx_uvicorn`
- [ ] 三一环境覆盖代码时确认未用本地 `trans.py` 覆盖云桌面专用 `trans.py`
- [ ] 部署后验证 `/health` 接口
- [ ] 部署后验证 `/crawl` 接口能正常启动浏览器
