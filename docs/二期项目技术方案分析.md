## 一、项目概述

### 1.1 项目定位

二期项目是一个基于 **browser-use AI Agent + LLM实时决策** 的智能招投标信息采集系统，采用AI驱动架构替代传统的规则驱动爬虫，实现无需维护选择器的自适应采集。

### 1.2 技术栈

| 组件       | 技术选型             | 版本/说明                    |
| ---------- | -------------------- | ---------------------------- |
| 核心框架   | browser-use          | AI Agent浏览器自动化框架     |
| 浏览器引擎 | Playwright + Chrome  | 内置于browser-use            |
| AI决策     | ChatBrowserUse       | LLM实时理解页面、做出决策    |
| 字段提取   | LLM + YAML配置       | 可配置化的字段提取           |
| 配置管理   | YAML                 | sites_config / extract_fields |
| 异步框架   | asyncio              | Python异步编程               |
| 日志系统   | Loguru + Rich        | 统一日志 + 进度条显示        |

## 二、系统架构

详细见 [二期项目流程图](二期项目流程图.mmd)

**数据流转**

```plain
main.py启动 → 加载YAML配置 → AI Agent接管浏览器
    → LLM理解页面 → 实时筛选条目 → 风控检测
    → 详情页处理（截图+字段提取） → 保存结果
```

## 三、核心模块分析

### 3.1 主入口 main.py

**文件位置**：`main.py`

**核心功能**：

- 加载网站配置（sites_config.yaml）
- 遍历网站列表，逐个处理
- 异步并发控制
- Rich进度条显示

**关键代码**：

```python
async def main():
    config = load_config('sites_config.yaml')
    for site in config['websites']:
        await process_site(site)
```

### 3.2 网站处理器 site_processor.py

**文件位置**：`src/site_processor.py`

**核心功能**：

- 创建浏览器实例（browser-use）
- 处理登录逻辑（如需要）
- 风控检测与自适应处理
- 调用列表页处理器

**设计亮点**：

- 统一的浏览器生命周期管理
- 智能登录处理（根据配置自动判断）
- 风控检测后自动跳过，不死循环重试

### 3.3 列表页处理器 list_processor.py

**文件位置**：`src/list_processor.py`

**核心功能**：

- AI扫描列表页，理解条目含义
- 实时筛选符合条件的条目
- 控制翻页逻辑
- 结构化输出（Pydantic模型）

**关键代码**：

```python
class ProcessResult(BaseModel):
    saved_count: int = Field(description="保存的条目数量")
    pages_processed: int = Field(description="处理的页数")
    titles: List[str] = Field(default_factory=list)
    risk_control: bool = Field(default=False, description="是否触发风控")

async def process_entire_site(browser, llm, filter_prompt, ...):
    agent = Agent(
        task=task,                          # 自然语言任务描述
        llm=llm,                            # LLM实例
        browser=browser,                    # 浏览器实例
        tools=tools,                        # 自定义工具
        output_model_schema=ProcessResult,  # 结构化输出
        max_steps=max_steps
    )
    result = await agent.run()
```

**设计亮点**：

- 使用Pydantic模型确保输出格式
- 自然语言描述任务，无需硬编码选择器
- 支持风控检测标记

### 3.4 详情页处理器 detail_processor.py

## 四、配置管理

### 4.1 网站配置

**文件位置**：`sites_config.yaml`

```yaml
websites:
  - name: 某央企采购平台
    url: https://example.com/tender
    login_required: false

  - name: 某国企招标网
    url: https://example2.com/bid
    login_required: true
    username: user123
    password: pass456
```

**配置项说明**：

| 字段           | 说明               |
| -------------- | ------------------ |
| name           | 网站名称（用于日志）|
| url            | 列表页URL          |
| login_required | 是否需要登录       |
| username       | 登录用户名（可选） |
| password       | 登录密码（可选）   |

### 4.2 字段提取配置

**文件位置**：`extract_fields.yaml`

```yaml
fields:
  - key: projectname
    name: 项目名称
    hint: 招标或者采购项目名称

  - key: budget
    name: 预算金额
    hint: 数字，以元为单位。如果是万元需要转换为元

  - key: winner
    name: 中标单位
    hint: 如果是中标候选人公告，返回排名第一的候选人
```

**配置项说明**：

| 字段 | 说明                         |
| ---- | ---------------------------- |
| key  | 字段标识（用于JSON输出）     |
| name | 字段中文名                   |
| hint | 提取提示（告诉LLM如何提取）  |

### 4.3 全局规则配置

**文件位置**：`src/prompts.py`

包含Agent的全局行为规则，如：
- 标签页操作规范
- 风控检测规则
- 禁止操作列表

## 五、核心特性

### 5.1 AI语义理解

**传统方式**（一期）：
```python
items = response.css('div.list-item')  # 硬编码选择器
```

**AI方式**（二期）：
```python
task = "找到标题包含'挖掘机'的招标条目，点击进入详情页"
agent = Agent(task=task, llm=llm, browser=browser)
```

**优势**：网站改版不影响采集，LLM自动适应页面变化

### 5.2 实时筛选

- 边浏览边筛选，不爬取无效数据
- 根据标题语义判断是否符合条件
- 日期过滤在浏览时完成

### 5.3 风控检测

```python
class ProcessResult(BaseModel):
    risk_control: bool = Field(default=False, description="是否触发风控")
```

- LLM自动识别验证码、访问限制等风控页面
- 触发风控后记录并跳过，不死循环重试
- 返回结果中标记风控状态

### 5.4 配置化扩展

新增网站只需：
1. 在 `sites_config.yaml` 添加配置
2. 运行测试

**无需编写代码**
